#Kioptrix: Level 1.2 (#3)

https://www.vulnhub.com/entry/kioptrix-level-12-3,24/

I didn't find this box easy at all, mainly because I wasn't searching/checking the right things. There's 3 ways in, 2 aren't so obvious and the other is brute forcing ssh - something you should probably be doing anyway once you've found some usernames.

0) Find the VM's IP


To find your VM's IP address, set your VM to ‘bridged’ so that it gets assigned an IP on your internal network and then use netdiscover to scan your network: netdiscover -r <internal network>/24

netdiscover -r 192.168.1.1/24
192.168.1.186 is my Kioptrix Level 1.2 box.
Take a snapshot of the VM at this point, just to be safe.

2) Local File Inclusion


I ran nikto to check the site for vulns, gobuster to bruteforce URLs, skipfish to crawl the site and uniscan to check for local file inclusion, as well as browsing through the site myself.

nikto -h http://192.168.1.186 -o nikto.txt

gobuster -u http://kioptrix3.com -w /usr/share/wordlists/dirb/common.txt -o gb-common.txt

gobuster -u http://kioptrix3.com -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -o gb-medium.txt

mkdir skipfish1 ; skipfish -o skipfish http://kioptrix3.com

uniscan -u http://kioptrix3.com/ -qweds

Trawling through all of the results, the only bit of info that became any use to me was the local file inclusion that both uniscan and skipfish found:
http://kioptrix3.com/index.php?system=/../../../../../../../../etc/passwd%00en

This will give you 2 usernames: dreg and john

4) CompanyPolicy


I checked the /home/loneferret directory before doing anything else (thankfully).
ht is a hex editor and feels pretty archaic. I had no idea what it was until googling it. F10 wouldn't work for me, I used CTRL+C to exit the program instead. You'll also to have set your TERM to have it display properly.

We have sudo permissions to run it, meaning that we can run ht as root.

This is a hex editor, which edits files, and it's running as root. i.e. we can edit any file on the system.

If you can edit any file on the system as root, you have 2 options to get a root shell- edit /etc/sudoers/ or /etc/shadow. The easier options is /etc/sudoers, where we'll give loneferret sudo privileges on everything; we'll be able to run every command as root.
