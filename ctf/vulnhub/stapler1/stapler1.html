https://www.vulnhub.com/entry/stapler-1,150/

This is my route through Stapler. There are apparently 2 ways to get a shell and 3 ways to get root. There's a lot of rabbit holes throughout this box, and it felt like I fell for all of them.

0) Find the VM's IP


To find your VM's IP address, set your VM to ‘bridged’ so that it gets assigned an IP on your internal network and then use netdiscover to scan your network: netdiscover -r <internal network>/24

netdiscover -r 192.168.1.1/24
192.168.1.237 is my Stapler box.
Take a snapshot of the VM at this point, just to be safe.

nmap -sC -sV -O -oN nmap/initial.txt 192.168.1.237

• 21/ftp allows for anonymous login (but no directory listing allowed). You could also bruteforce.
• 22/ssh is to be noted down for later.
• 53/dns there isn't much to do with on a vm.
• 80/http reports a php cli server(?) - so probably isn't a usual site (development perhaps?). Nikto/gobuster it.
• 139/samba can be used to gather usernames.
• 666/doom? looks like it contains a hidden message.
• 3306/mysql might allow remote login.
• 12380/http looks like a more usual web server.

I looked at all of these, trying to gather information from each and deciding where to go next from there.
If you nikto scan the web server on 12380 you'll find that it's actually a HTTPS site. Despite this, the site wouldn't work for me. I think it was a VM issue, it was redirecting to a non-existent internal URL ¯\_(ツ)_/¯
