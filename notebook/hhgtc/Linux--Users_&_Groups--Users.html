# Linux - User Management - Users


Every userland process has an owner (whomever started the process) and the process runs with the permissions of that owner.
Users can edit and modify their own processes, but can't tamper with other users' processes.

## 3 Types of Users


### superusers


◇ i.e. root
◇ root is the classic superuser
◇ root can edit/modify/run anything it wants - it's basically the adminstrator account.

### system services


◇ apache, mysql, fail2ban, tcpdump
◇ these are accounts for services and system daemons
◇ creating a user for a service helps manage their permissions
▪ e.g. Apache needs privileged access for some functionality - like opening privileged ports 80 and 443
▪ but shouldn't run as root because that would be a security issue
▪ creating a specific user for the service helps manage permissions/security

### human users


◇ accounts that everyday users/humans will use - Bob, Jane, Harry etc.
◇ these accounts will (almost) all have their own home directory where they can store files - /home/<username>

## UIDs - User IDs


• All users have a UID - User ID - assosciated with them, which is what the kernel uses to manages users
◇ Bob = 1001, Jane = 1002 etc.
• UIDs below 500 are reserved for root and system services
