# WinRM


https://docs.microsoft.com/en-gb/windows/win32/winrm/portal?redirectedfrom=MSDN

WinRM stands for Windows Remote Management.
It's a Microsoft protocol that allows for remote management of Windows machines over HTTP(s) using SOAP.
• WinRM's backend uses WMI.
• SOAP communicates over ports 5985 (HTTP) and 5896 (HTTPS)

If WinRM is enabled on a machine, you can remotely administer the machine from PowerShell.

The easiest way to to tell if WinRM is enabled is if ports 5895 and 5896 are open.
• Not often enabled on workstation/client PCs -> but common on Domain Controllers

## Port 5985 (HTTP) or 5986 (HTTPS) - wsman


• SOAP based protcol - Simple Object Access Protocol
• Encrypted by default
• Is the official remote management protocol in Windows 2012 R2 and higher
