## GET and POST Data


Good example in tut03_get_and_post.py
CherryPy abstracts GET and POST data away.
You set the parameters you want in the page definition, e.g:
def recon(self, os=None, arch=64, comp_name=None):

And can then grab the parameters from GET or POST requests, it doesn't matter.

e.g. GET
http://127.0.0.1:8080/recon?os=Windows10

e.g. Barebones POST
Host: 127.0.0.1:8080
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:85.0) Gecko/20100101 Firefox/85.0
Content-Type: application/x-www-form-urlencoded

os=Windows10Pro&arch=32
