# GET and POST


• before the browser sends info, the URL gets encoded
◇ spaces are replaced with +
◇ non-alphanumeric characters are replaced with hex values
• the page & encoded information are separated by a ?

## GET


GET sends the URL encoded data appended to the end of the page request
http://test.com/index.html?name1=john&name2=bob

Access the GET data using QUERY_STRING environment variable
Use $_GET associate array to access all sent the info in the GET request

• NOT GOOD FOR PASSWORDS OR SENSITIVE INFORMATION
• produces a long string that appears in your server logs & Location: header
• limited to 1024 characters
• can't send binary data

## POST


POST sends data via HTTP headers.

Access data using QUERY_STRING environment variable
Use $_POST associative array to access all data sent via POST

• no restriction on size of data
• can send ASCII & binary data

Because data goes through HTTP headers, the security depends on HTTP / HTTPS

## The $_REQUEST Variable


$_REQUEST stores the contents of $_GET, $_POST, and $_COOKIE
If you want both GET and POST data, use $_REQUEST

<?php
   if( $_REQUEST["name"] || $_REQUEST["age"] ) {
      echo "Welcome ". $_REQUEST['name']. "<br />";
      echo "You are ". $_REQUEST['age']. " years old.";
      exit();
   }
?>
