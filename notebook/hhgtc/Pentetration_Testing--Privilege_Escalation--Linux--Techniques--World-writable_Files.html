# World-writable files


World-writable files might be useful in some way. World-writable files owned by root might be extremely useful - you can add your own malicious code into the file.

Examples
• hackthebox Solidstate

## Tips


• If you ever find any file that is owned by root and that you can modify, edit it! (create a backup first)
◇ i.e. to send a reverse shell back to you, and leave a listener running. You never know, there might be some random cronjob running that file.

## Find


Find world-writable directories (also stolen from g0tm1lk)
find / -xdev -type d \( -perm -0002 -a ! -perm -1000 \) -print 2>/dev/null
Find world-writable files (modified of above)
find / -xdev -type f \( -perm -0002 -a ! -perm -1000 \) -print 2>/dev/null

### g0tm1lk


Find world writable directories
find / -writable -type d 2>/dev/null
find / -perm -222 -type d 2>/dev/null
find / -perm -o w -type d 2>/dev/null

Find world executable folders
find / -perm -o x -type d 2>/dev/null

Find world writable & executable folders
find / \( -perm -o w -perm -o x \) -type d 2>/dev/null
