## runas


https://ss64.com/nt/runas.html
Execute a program under a different user account (non-elevated).

### Consider


• REQUIRES AN INTERACTIVE SESSION
◇ because you will be prompted for the password
◇ because spawning cmd will start a new process / open a new window
• whoami will always return previous user. I think commands run with other user's privileges, but you're still sat inside original user's instance

### Options


/netonly Use the credentials for remote access only
/noprofile Do not load the user's profile.

• /profile is not compatible with /netonly

### Usage


runas /netonly /user:DOMAIN\username "cmd.exe"
runas /noprofile /netonly /user:DOMAIN\username cmd.exe

To read result of single command, output the result of command to text file
(file location has to be writable by user you're runas'ing)
runas /netonly /user:TOKYO\spencero "cmd.exe /c whoami > C:\Users\Public\output.txt"
