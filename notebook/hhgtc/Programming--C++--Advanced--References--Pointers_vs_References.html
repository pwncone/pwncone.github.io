#Pointers vs References


The decision whether to use a pointer or a reference is determined by the program requirements.
C++ programmers generally prefer to use references, because they're easier to use and understand than pointers.
However, there's certain rules:

RuleReferencesPointers
Can be declared without initialisationNoYes
Can be reassignedNoYes
Can contain a 0 (null) valueNoYes
Easy to useYesNo


The choice between using a reference or a pointer follows these guidelines:
• If you don't want to initialise in the declaration - pointer
• If you want to be able to reassign another variable - pointer
• Otherwise, always use a reference
