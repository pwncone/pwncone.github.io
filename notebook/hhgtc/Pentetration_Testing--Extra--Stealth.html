# A word on stealth


Stealth is about blending in, and not leaving a trail. That's it.
It's a lot about understanding the current context that you're in hiding your attacks behind commonplace activities.

• Use commonplace sysadmin tools rather than known hacking tools (if you can avoid it)
• Avoid dropping files to disk (deploy your own SMB share, remotely mount disk via RDP, etc.)
• Leave persistence where it makes sense
