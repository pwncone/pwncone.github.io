#Storage Classes


A storage class defines the scope (visibility) and life-time of variables and/or functions within a C++ Program

static Storage Class


By default, variables are local to the scope in which they are defined.
Variables can be declared as static to increase their scope (function) up to the file containing them. As a result, these variables can be accessed anywhere inside a file.

Static local variables maintain their values between function calls

(this is C example code, but the same still applies)
e.g. in this code count will be intialised and deleted when the runner() functions
the output will be 1 1
#include <stdio.h>

int runner() {
    int count = 0;
    count++;
    return count;
}

int main()
{
    printf("%d ", runner());
    printf("%d ", runner());
    return 0;
}


but here, because of the value of count gets saved after the first runner() function has complete because we're using a static variable, the output will be 1 2
#include <stdio.h>

int runner() {
    static int count = 0;
    count++;
    return count;
}

int main()
{
    printf("%d ", runner());
    printf("%d ", runner());
    return 0;
}



register Storage Class


The register storage class is used to define local variables that should be stored in a register instead of RAM
This means that the variable has a maximum size equal to the register size (usually one word)
Registers should only be used for variables that require quick access such as counters

register int miles;


Using register means that the variable only MIGHT be stored in a register depending on hardware and implementation restrictions (i.e. it won't definitely use a register)
