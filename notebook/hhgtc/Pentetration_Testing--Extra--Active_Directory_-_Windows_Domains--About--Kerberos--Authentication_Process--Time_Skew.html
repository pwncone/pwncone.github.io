# Kerberos Time Skew


https://docs.microsoft.com/en-us/windows/security/threat-protection/security-policy-settings/maximum-tolerance-for-computer-clock-synchronization
Kebreros' time skew is a security setting to mitigate replay attacks (an attacker using an day/week/month old Kerberos ticket to gain access to the network at a later time).

When Kerberos is checking requests/issuing tickets, it checks the time skew.
Because individual computer clocks within a domain will never be in sync, there's a maximum tolerance setting.
The default time skew is 300 seconds, or 5 minutes.
This makes time synchronisation between computers within an Active Directory domain vital.
