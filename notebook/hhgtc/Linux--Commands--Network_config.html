# Linux - Commands - Network config



## ifconfig


ifconfig - view network interfaces
◇ eth0 = first Ethernet card in the machine
◇ wlan0 = wireless interface
◇ lo = loopback interface

ifconfig <interface> down - disable interface
ifconifg <interface> up - enable interface

ifconfig eth0 192.168.2.1 netmask 255.255.255.0 up - assign IP address and netmask to eth0 and bring it up

## ip


ip link show - show interface information for all interfaces
ip -s link show eth0 - show the statistics of an interface
ip address show - show ip addresses allocated to interfaces
ip link set eth0 up - bring interface up
ip link set eth0 down - bring interface down
ip address add 192.168.1.1/24 dev eth0 - add an IP address to an interface

ip neighbour show - view arp cache

## route


sudo route add -net 192.168.2.1/23 gw 10.11.12.3 - add a new route
sudo route del -net 192.168.2.1/23 - delete a route
ip route add 192.168.2.1/23 via 10.11.12.3 - add a route

delete a route
ip route delete 192.168.2.1/23 via 10.11.12.3
or
ip route delete 192.168.2.1/23

## arp


• The ARP cache is empty when a machine boots up
• It gets populated as packets are sent to other hosts

arp -a - view arp cache

## dhclient


sudo dhclient - obtain a fresh IP

## Network Manager


• Most distributions utilize the NetworkManager daemon to configure their networks automatically
• It manages your network's hardware and connection information

nm-tool
nm-tools reports NetworkManager's state and its devices

nmcli
The nmcli command allows you to control and modify NetworkManager (if you don't want to do it through the GUI)
