# Linux - Partitioning - How to Partition Disks


• Partitioning a hard drive divides the available space into sections that can be accessed independently
• Parition information is stored in a partition table (e.g. MBR or GPT)
• Partition tables are created with tools (e.g. fdisk, gdisk, parted)

There are multiple linux tools to parition disks:
• fdisk - supports MBR but not GPT
• gdisk - same as fdisk, but supports GPT and not MBR
• parted - supports both MBR and GPT partitioning
• gparted - a GUI version of parted

After partitioning, you format the partition with a filesystem (e.g. ext4, NTFS etc.)

Standard BIOS/MBR Partition Table layout:


/ /dev/sda1 Linux Boot flag 23-32 GiB
[SWAP] /dev/sda2 Linux swap ~20% of RAM
/home /dev/sda3 Linux Remainder of device

Standard UEFI/GPT Partition Table layout:


/boot /dev/sda1 EFI System Partition 550 MiB
/ /dev/sda2 Linux root 23-32 GiB
[SWAP] /dev/sda3 swap ~20% of RAM
/home /dev/sda4 Linux /home Remainder of device
(or ditch /home and just make / the remainder of the device)

## parted


1. Launch parted - sudo parted
2. Select the device you want to parition - select /dev/sdb2
3. View the current partition table - (parted) print

(parted) print
Model: Seagate (scsi)
Disk /dev/sda: 21.5GB
Sector size (logical/physical): 512B/512B
Partition Table: msdos

Number  Start   End     Size    Type      File system     Flags
 1      1049kB  6860MB  6859MB  primary   ext4            boot
 2      6861MB  21.5GB  14.6GB  extended
 5      6861MB  7380MB  519MB   logical   linux-swap(v1)
 6      7381MB  21.5GB  14.1GB  logical   xfs


4. Partition the device - mkpart primary <start> <end> e.g. mkpart primary 500 2500
The start and end points are where the partitions take up space on the hard drive.

Resize a parition - resize <partition number> <start> <end> e.g. resize 2 200 4000
