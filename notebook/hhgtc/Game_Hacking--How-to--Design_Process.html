# Design Process


Just my throughts on the process of designing cheats

## Internal


• Inject successfully into the game (spawn a message box)
• Get window size (directX?) / game resolution (openGL?)
• Hook buffer that runs every frame. Get this to work without the game crashing
• Draw a box, a line, and text in-game
• Draw a box/line/text where you want to in-game (make sure the screen coordiantes draw where you say)
• Loop through the players in game and write data to console to verify (name/stats/whatever/whole struct)
• Get world to screen working (in a console, print TRUE if enemy on-screen, FALSE if enemy off-screen)
• Start with basic ESP - print text over enemies OR create snaplines (both are an equal step imo)
• Draw a box around enemy

At this point you're pretty comfortable doing whatever.
Implement triggerbot, more advanced ESP, aimobt; whatever you feel like.
You know your code works now so you can just about do anything and debug it if it goes wrong.
