# x86 Data Sizes


### The Confusion with word Size


https://stackoverflow.com/questions/55430725/whats-the-size-of-a-qword-on-a-64-bit-machine
The sizes of bits and bytes stay consistent throughout instruction sets.
However, the point at which data sizes can become confusing is the word size.

word typically refers to the size of the instruction set - e.g. 16 bit, 32 bit, etc.
However, instruction sets have evolved over time (from 16 bit, to 32 bit, to 64 bit).
Therefore, to prevent having to modify instructions, the word size is kept the same as it was originally.

In x86, architecture evolved out of the old 16-bit 8086 architecture.
Therefore, a word is 16 bits.

For most other instruction sets, they started out as 32 bit ISAs (AArch64, MIPS64, PowerPC64, etc.)
Therefore, a word on those architectures is 32 bits.

For x86, a word is 16 bits.
The best way to find the word size is to read the architecture's ISA manual.

### x86 Data Sizes


BIT
A bit is the smallest possible piece of data - it's either a 1 or a 0
When you put bits together, you get the binary number system
• 00000001 = 1
• 00000010 = 2
• 00000011 = 3
• etc.

BYTE
A byte is 8 bits put together (like in the example above)
8 bits can represent the number range of 0 - 255

WORD
A word is 2 bytes - 16 bits.

DWORD
DWORD is short for Double Word
A dword is 4 bytes - 32 bits

QWORD
QWORD is short for Quad Word
A qword is 8 bytes - 64 bits
These only exist on x86-64.
