#Macro Functions


#define can also be used to create macro functions that will be substituted in the source code before compilation.

#define HALF(n) (n/2)

// macro functions
#define SQUARE(n) (n*n)
#define CUBE(n) (n*n*n)

#include <iostream>
using namespace std;

// inline functions
inline int square(int n) {return (n * n);}
inline int cube(int n) {return (n * n * n);}

int main()
{
    // macro functions test
    cout << "--- MACRO FUNCTIONS TEST ---" << endl;
    cout << SQUARE(5) << endl;
    cout << CUBE(5) << endl;
    
    cout << endl;

    // inline functions test
    cout << "--- INLINE FUNCTIONS TEST ---" << endl;
    cout << square(5) << endl;
    cout << cube(5) << endl;
    
    return 0;
}


Inline vs Macro Functions
Unlike regular C++ functions, macro functions don't perform any type checking (so it's easy to create a macro function that causes errors).
Because of this, inline functions are preffered over macro functions.
But, because macro functions are substituted in and avoid the overhead of a function call - macro functions will run faster.
