# Linux - Permissions - Changing file permissions


chmod is used to change permissions of files.
chmod - short for change mode.

## chmod


With chmod, you can use either the letters format or the octal format to change the permissions of a file:

### Letters format


u = user (owner)
g = group
o = others

+ = give permissions
- = remove permissions

Example:
chmod ug+x <file/directory>
Give execute permissions to the user and the group

Think of it like ‘giving the user write permissions’
or ‘taking execute permissions away from the group’ (chmod g-x <file>)

### Octal format


(This is binary, which is why it goes 1, 2, 4)
4 = r (read permissions)
2 = w (write permissions)
1 = x (execute permissions)

Add up the numbers to what you want e.g.
7 = read/write/execute (4 + 2 + 1 = 7)
6 = read/write (4 + 2 = 6)
5 = read/execute (4 + 1 = 5)

Example:
chmod 777 <file/directory>
(read/write/execute permissions for everyone)

1st number = user/owner permissions
2nd number = group permissions
3rd number = other's permissions

To set permissions for a directory + all files in that directory, use -R (recursive)
chmod -R 770 <directory>
