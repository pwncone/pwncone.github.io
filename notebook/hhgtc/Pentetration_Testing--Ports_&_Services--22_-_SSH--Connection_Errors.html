# Connection Errors


Sometimes connection errors are part of CTFs, and sometimes you just run into them for no reason.

## No matching key exchange method found


• The server should response with their own key exchange methods.
• If talking to OpenSSH 7 or higher, you can specify the key exchange method on the command line with -okexAlgorithms=+

ssh -okexAlgorithms=+<offered algorithm> user@10.10.10.14

• If talking to versions lower than OpenSSH 7, specify the key exchange algorithm in ~/.ssh/config
KexAlgorithms diffie-hellman-group-exchange-sha1
ssh bob@10.10.10.14

## No matching cipher found


• The server should respond with their offer of ciphers
• Specify the cipher to use in ~/.ssh/config
• Source
◇ https://forum.manjaro.org/t/cant-connect-to-ssh-host/33332/2

Ciphers aes128-cbc
ssh bob@10.10.10.14

## Skipping ssh-dss key <key> - not in PubkeyAcceptedKeyTypes


• https://unix.stackexchange.com/questions/247612/ssh-keeps-skipping-my-pubkey-and-asking-for-a-password/247614

Modern SSH clients (what you're probably running), won't accept DSA keys anymore.
To force SSH to use the DSA key, use -o PubkeyAcceptedKeyTypes=+ssh-dss.

ssh -o PubkeyAcceptedKeyTypes=+ssh-dss -i f1fb2162a02f0f7c40c210e6167f05ca-16858 bob@10.11.1.136

## Sign_and_send_pubkey: no mutual signature supported


Is also probably an issue with using an old key type.
Use -o PubkeyAcceptedKeyTypes=+<type> to solve. (not 100% on this one. Google it).
