#Casting pointers


https://stackoverflow.com/questions/17260527/what-are-the-rules-for-casting-pointers-in-c/17260931 - fairly decent explanation

Define a character array - char* passcode = ‘aaaabbbb’
Cast the char* pointer to an int* pointer - int* integerPasscode = (int*) passcode;
→ (int*) passcode will retrieve the data pointed to by passcode and treat it as an integer

Explanation:
◇ passcode is a char pointer - it points the start of our string (e.g passocde = ‘aaaabbbb’, p points to ‘a’)
◇ (int*) passcode will retrieve the data pointed to by passcode and treat it as an integer
▪ e.g. passcode = ‘aaaabbbb’, (int*) passcode will retrieve that data as 61616162626262 (not ‘aaaabbbb’)
▪ 61 hex = 97 decimal = 'a' in ascii, 62 hex = 98 decimal = 'b' in ascii

◇ int* integerPasscode is an integer pointer
▪ which means that int* integerPasscode = (int*) passcode points to '6161616162626262'
◇ but because integers are 4 bytes long, int* integerPasscode only points to 4 bytes at a time
▪ e.g integerPasscode[0] points to the first 4 bytes - 61616161, integerPasscode[1] points to the second 4 bytes - 62626262
