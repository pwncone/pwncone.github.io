# Linux - Boot Process


The Linux boot process can be broken down in 4 simple stages:

## 1. BIOS/UEFI


• BIOS stands for Basic Input/Output System
• UEFI stands for Unified Extensible Firmware Interface
• The BIOS/UEFI initializes the hardware
◇ hard disks
◇ keyboard
◇ etc.
• And then checks the hardware is ok with a POST - Power-On Self Test
• The BIOS/UEFI then loads the bootloader
• BIOS is older, UEFI is newer. They both load the bootloader a little differently

## 2. Bootloader


• The bootloader is responsible for loading the kernel into memory
• It then starts the kernel and transfers control to it (for it to run init/systemd/startup scripts)
• GRUB is the most common Linux bootloader
◇ can be used with BIOS and UEFI

## 3. Kernel


• The kernel initializes devices and memory
• And starts the the init process (startup scripts)

## 4. Init


• Init starts and stops essential services on the system
• It reads the /etc/inittab file and runs script to set up the system for use
• There are 3 major implementations of init in Linux distributions
◇ System V (sysv)
◇ Upstart
◇ Systemd
