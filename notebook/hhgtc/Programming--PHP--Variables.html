# Variables


$variable = 10;
$textvar = "hey";

• denoted with $
• assigned with =
• can, but don't need to, be declared before assignment
◇ $variable = 10; - fine
◇ $variable; $variable = 10; - also fine
• if variable is used before its assigned, it will have a default type
• no intrinsic types - don't need to declare a variable's data type

## Naming


• must begin with letter or underscore
• can use letters/numbers/underscores
• can't use special chars:
◇ - % ( ) . & ,

## Scopes


• local
• global
• static
• function parameters

## Data Types


int


$int_var = 12345;
$int_var = 12345 + 23;

double (floating)


$my_double = 2.234;

bool


PHP provides constants:
if (TRUE)
   print("This will always print<br>");
else
   print("This will never print<br>");


• for ints, 0 = false
• for strings, empty string = false
• NULL is always false
• for arrays, no ther values = false
• valid resources = true
• doubles + booleans are a bad combo, don't use

$true_num = 3 + 0.14159;
$true_str = "Tried and true"
$true_array[49] = "An array element";
$false_array = array();
$false_null = NULL;
$false_num = 999 - 999;
$false_str = "";


NULL


$my_null = NULL;
$my_null = null; - lowecase also viable

string


valid strings:
$string_1 = "This is a string in double quotes";
$string_2 = 'This is a somewhat longer, singly quoted string';
$string_39 = "This string has thirty-nine characters";
$string_0 = ""; // a string with zero characters


single quotes are (mostly) interpretted literally
double quotes preprocess included PHP code - e.g. interprets character sequences and evaluate variables

// "name" won't be printed because using single quotes (literal interpretation)
$variable = "name";
$literal  = 'My $variable will not print!';
print($literal);

// "name" will be printed because double quotes evaluate variable
$variable = "name";
$literal  = "My $variable will not print!<br>";
print($literal);


• no limits on string length apart from memory limitations
