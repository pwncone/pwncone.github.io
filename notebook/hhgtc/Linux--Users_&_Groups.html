# Linux - Users & Groups


• Users and groups exist solely for access and permissions
• When running a process, it will run as the owner of that process whether that is Jane or Bob
• Each user has their own home directory where their user specific files get stored, this is usually located in /home/username

## User IDs


• The system uses user ids (UID) to manage users
◇ Usernames are the friendly way to associate users with identification
◇ But the system identifies users by their UID

## root


• root is the most powerful user on the system
◇ root can access any file and start and terminate any process.
• For that reason, it can be dangerous to operate as root all the time

## sudo


• As a safeguard against the powers of root
◇ a user can run a single command as root with sudo
• sudo stands for superuser do

## Groups


• The system also uses groups to manage permissions
◇ Groups are just sets of users with permissions set for that group,
• They are identified by the system from their group ID (GID)
