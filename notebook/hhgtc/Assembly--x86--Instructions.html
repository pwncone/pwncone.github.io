# Instructions


Assembly language is made up of instructions.
Instructions tell the CPU to do stuff.

An instruction is made up of:
• mnemonic - an instruction that the CPU can execute e.g. mov
• operands - the data for the CPU instruction to manipulate e.g. ecx 0x42
◇ There are 3 types of operand:
▪ Immediate values
- fixed values e.g. 0x42
▪ Registers
- eax, ebx, ecx, etc.
▪ Memory addressing
- denoted using brackets e.g. [eax]
→ e.g. [edx] references the data in the memory address that edx currently stored
→ mov eax, [edx] - move data from memory address stored in edx to eax
→ mov eax, [0x0437C4] - move data from memory address 0x4037C4 into eax
→ mov eax, [ebx+esi*4] - calculate ebx+esi*4 and use result as a memory address. move data from that memory address into eax

Here's an instruction that moves the value 0x42 into the EAX register:
mov ecx 0x42


## 2 types of Syntax


There are 2 types of syntax for x86:
• IA-32 / Intel syntax (commonly used on Windows) - has the destination 1st and the source 2nd
• AT&T syntax (commonly used on Linux) - has the source 1st and the destination 2nd

IA-32 / Intel Syntax
mov ecx 0x42
Mnemonic Desination operand Source operand
mov ecx 0x42

AT&T Syntax
mov 0x42 ecx
Mnemonic Source operand Destination operand
mov 0x42 ecx


## Opcodes


Opcodes are the hex forms of the human-readable instructions
e.g the opcode string for mov ecx, 0x42 is B9 42 00 00 00

Instruction: mov ecx, 0x42
Opcodes: B9 42 00 00 00


## Types of Instructions


There are 5 categories of instructions:
• Data movement - move data from here to there
• Data access - load data from here
• Arithmetic - add, subtract, multiply, divide
• Logic - and, or, xor
• Control flow - jump, push data, pop data, call

## Common Instructions


mov - data movement
lea - data access
add, sub - arithmetic
or, and, xor - logic
jmp, jne, jnz, jnb - control flow
push, pop, call, leave, enter, ret - control flow
