# Linux - Permissions - Process Permissions


• There are three UIDs associated with every process
◇ effective user ID - has the same permissions as the user or group that started the process
▪ used to grant access rights to a process
◇ real user ID - the ID of the user that launched the process
▪ used to track down who the user who launched the process
◇ saved user ID - allows a process to switch between the effective UID and real UID
▪ useful because we don't want our process to run with elevated privileges all the time

## Example


Your real user ID is 500.
Your effective UID is the same as your user ID - 500.

Sally's user ID is 600.

You run passwd.

The passwd binary has a SUID bit on it, which means that when you run it, your effective user ID is 0 (root).
User ID = 500
Effective UID = 0

You try to modify Sally's password.
Your real user ID prevents you from changing Sally's password, because passwd knows (by reading your real user ID), that you're not Sally.
