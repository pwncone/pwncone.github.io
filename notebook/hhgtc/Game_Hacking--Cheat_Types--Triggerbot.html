# Triggerbot


There's 2 components to a triggerbot.
First, you have to check if you're aiming at a player.
Second, you have to shoot.

Great overview here:
https://gamehacking.academy/lesson/26

## Check if aiming at player


### Crosshair Entity


When you hover over an enemy, most games will store that enemy somewhere.

## Shoot


### Set shooting to TRUE in player struct


If the player struct has an isShooting value, set that to TRUE

// Check if our crosshair is on a player
Entity* entity = d_GetCrosshairEntity();

// If not already pressing shoot button
if (!GetAsyncKeyState(VK_LBUTTON))
{
// If aiming at valid enemy, shoot for them
if (entity != NULL)
{
if (entity->team != player->team)
player->is_attacking = true;
}
// If not aiming at eney, stop shooting
else
{
player->is_attacking = false;
}
}


### Send MOUSEEVENTF_LEFTDOWN Event


If you're aiming over the enemy,
you could send a left mouse down event to shoot.

INPUT input = { 0 };
if (on_enemy == TRUE) {
    input.type = INPUT_MOUSE;
    input.mi.dwFlags = MOUSEEVENTF_LEFTDOWN;
    SendInput(1, &input, sizeof(INPUT));
}
else {
    input.type = INPUT_MOUSE;
    input.mi.dwFlags = MOUSEEVENTF_LEFTUP;
    SendInput(1, &input, sizeof(INPUT));
}


### Use console commands


Somtimes in the game there's console commands that will make the player shoot
e.g. in CoD4 there's /+attack to shoot and /-attack to stop shooting

You could try writing a command into the console to make your player shoot.
