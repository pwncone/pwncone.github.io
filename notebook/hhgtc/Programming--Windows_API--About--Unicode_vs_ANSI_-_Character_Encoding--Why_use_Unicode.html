Always use Unicode
• Improves execution performance & uses less memory
◇ Windows internally does everything with Unicode characters and strings, so when you pass an ANSI character or string, Windows must allocate memory and convert the ANSI character or string to its Unicode equivalent
• Your application will support multiple languages
• Helps when interoperating with COM and .NET framework

Windows uses UTF-16
• UTF-16 encodes each charctaer as 2 bytes (16 bits)
• Characters from most languages can be represented via a 16bit value
• The .NET framework always encodes characters & strings in UTF-16
◇ so using UTF-16 in your Windows application will improve performance and reduce memory consumption if you need to pass characters or strings between native and managed code
