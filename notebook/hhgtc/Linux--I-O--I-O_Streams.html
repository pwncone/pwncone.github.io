# Linux - I/O Streams


Processes use I/O streams to receive input and return output.
I/O redirection allows you to change where programs receive input and where they output to.

There 3 I/O streams:
• stdout (standard out)
• stdin (standard in)
• stderr (standard error)
◇ error outputs
◇ e.g.
▪ list a fake directory - ls /fake/directory
▪ stderr - ls: cannot access /fake/directory: No such file or directory

## Redirection Operators


### stdout


• > directs stdout to where it's pointing (and overwrites data if it's already there)
• >> appends stdout to where it's pointing

### stdin


• < takes input from files or other processes
◇ cat < peanuts.txt > banana.txt
◇ ls < peanuts.txt > banana.txt

### stderr


• To redirect stderr you have to use its file descriptor - 2
◇ Direct stderr to text file - ls /fake/directory 2> error.txt
◇ Direct stderr and stdout to text file - ls /fake/directory > peanuts.txt 2>&1
◇ Direct stderr to /dev/null to stop seeing errors - ls /fake/directory 2> /dev/null
