# JSON


JSON - JavaScript Object Notation - is a standardised syntax for storing and reading data.
Because of its simplicity, it's popular in web apps and small-scale applications.
Sometimes you'll find web services that send JSON requests to retrieve data and then render them.

Links
• https://jsonplaceholder.typicode.com/ - Good site to test/learn about JSON and the REST API

### Request auth token by supplying username/password


Normally you submit a token to the REST API in order to authenticate.
If you don't have an auth token, you might be able to authenticate using a username and password.

curl --header "Content-Type: application/json" --request POST --data '{"username":"admin", "password":"Zk6heYCyv6ZE9Xcg"}' http://10.10.10.137:3000/login
◇ used in hackthebox Luke

### Submit auth token and retrieve data


If you have an auth token, you can supply it in the Authorization header:
Authorization: <auth type> <token>

## Retrieve Data


To retrieve data,
• you query an object (users)
• and then choose a field in that database (e.g. 5) OR search for a value of a parameter

For example:
https://jsonplaceholder.typicode.com/users/5 - query users object and list the 5th field
https://jsonplaceholder.typicode.com/users?username=Bret - query users object and search username parameter for Bret

### Get data using curl (no auth)


curl --header "Content-Type: application/json" --request GET http://10.10.10.137:3000/users

### Get data using curl (auth)


Different authentication methods:
curl -H "Authorization: OAuth Zk6heYCyv6ZE9Xcg" http://10.10.10.137:3000/login
curl -H "Authorization: Bearer Zk6heYCyv6ZE9Xcg" http://10.10.10.137:3000/login
curl -H "Authorization: Basic Zk6heYCyv6ZE9Xcg" http://10.10.10.137:3000/login
curl -H "Authorization: Token Zk6heYCyv6ZE9Xcg" http://10.10.10.137:3000/login
curl --anyauth --user root:Zk6heYCyv6ZE9Xcg http://10.10.10.137:3000/login
