## What is Kerberoasting?


There are 2 points in the Kerberos authentication process where NTLM hashes are being sent over the network
• in the AS-REP reply when requesting a TGT
• in the TGS-REP reply when requesting a ticket for a service

Kerberoasting is where you grab the NTLM password hashes from these 2 packets, extract them, and crack them.
This is what Kerberoasting is.

TGS_REP Kerberoasting was discovered by Tim Medin and debuted at DerbyCon in 2014.

### AS-REP vs TGS-REP


• AS-REP = attacking a user
• TGS-REP = attacking a service account

If you kerberoast an AS-REP, you're kerberoasting a user.
This is because the AS-REP reply hashes the TGS session key it returns with the user's NTLM hash who requested the ticket.

If you kerberoast a TGS-REP, you're kerberoasting a service account.
This is because the TGS-REP reply hashes the TGS ticket it returns with the NTLM password hash of the requested service.
