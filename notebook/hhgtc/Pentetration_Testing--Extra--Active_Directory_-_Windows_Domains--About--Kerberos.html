## About


Kerberos was created in the 1980s by MIT.
The name comes from Cerberus, the mythical dog in Greek mythology that guards the gates to the Underworld.
Microsoft started using Kerberos in Windows 2000.

Kerberos is a network authentication protocol.
Users authenticate once with a central Kerberos server.
Upon authenticating succesfully, users are given a Key - which they can use to access other services on the network.
The Kerberos KDC - Key Distribution Center - runs on port 88.

Multiple OSs - Linux, Windows, etc. - can all talk to one kerberos server and work together providing they're all running the same Kerberos version.

### Port 88 - KDC -

Key Distribution Center


The KDC - Key Distribution Center - runs on port 88 (usually on the Domain Controller).
The KDC authenticates users and, if successful, grants them tickets.

• The KDC stores all of the secret keys for user machines and services in its own database - NTDS.dit
• Secret keys are passwords plus a salt that are hashed
- the hash algorithm is chosen during implementation of the Kerberos setup.
- For services or host machines, there are no passwords (who would enter it?)

Authentication Service - Authenticates users intitially onto the system
Ticket Granting Service - Provides users with tickets (which give access to servers/services on the network)

## Kerberos Tickets


In Kerberos there are 2 tickets:
• TGT - Ticket Granting Ticket
• TGS - Ticket Granting Service

TGT - Ticket Granting Ticket
This ticket allows a client (user) to request access to a TGS ticket.
(received during the authentication process)

TGS - Ticket Granting Service
This ticket allows a client (user) to access to a service within the network.
