#.h Header files


.h header files can contain other functions/code. They're a way of organising the code for you program.

add.h
Make the header file and write your code - add.h
inline int add(int x, int y) {return (x+y);}


addNumbers.cpp
Now the add() function can be used in another file if you #include “add.h”
#include "add.h"

#include <iostream>
using namespace std;

int main()
{
    cout << "9 + 3 = " << add(9, 3) << endl;
    
    return 0;
}



#Inclusion Guards


Problems occur if you #include the same header file in multiple files in your program.
The header file and its contents will be defined once when the compiler first comes across it, but on subsequent includes of the same file the compiler will fail (because the contents of the header has already been defined - causing conflicts.)

The (popular) solution to this problem is to use preprocessor directives - #ifndef
Name your label for #ifndef the same as the header filename but in all-caps and _H at the end
e.g. add.h → #ifndef ADD_H

If ADD_H hasn't been defined (i.e. the header file hasn't been defined yet), then run the following code in #ifndef.
Otherwise, if ADD_H has been defined already, ignore the header file and move on.

This solves the problem.

add.h with Inclusion Guard
// inclusion guard
// if ADD_H isn't defined - define it (this header file has no been checked)
// if ADD_H is already defined - ignore this header file and don't define it again
#ifndef ADD_H
    #define ADD_H
    inline int add(int x, int y) {return (x+y);}
#endif


tripleAdd.h - This is what would make compilation file if no inclusion guard because add.h has been included in 2 files
#include "add.h"

inline int tripleAdd(int x) {return add(x, add(x, x));}


addNumbers.cpp
#include "add.h"
#include "tripleAdd.h"

#include <iostream>
using namespace std;

int main()
{
    cout << "9 + 3 = " << add(9, 3) << endl;
    cout << "9 x 3 = " << tripleAdd(9) << endl;
    
    return 0;
}
