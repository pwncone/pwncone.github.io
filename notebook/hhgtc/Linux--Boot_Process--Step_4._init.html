#Linux - Boot Process - init


init first process that gets started and it starts all the other essential services on the system.
There are 3 major implementations of init in Linux:

## System V init (sysv)


• System V is the traditional init system.
• It sequentially starts and stops processes based on startup scripts.
• The state of the machine is denoted by runlevels,
◇ each runlevel starts or stops a machine in a different way

## Upstart


• Upstart is the init you'll find on older Ubuntu installations.
• Upstart starts jobs that perform certain actions in response to events

## Systemd


• Systemd is the new standard for init
◇ It's goal oriented
◇ Goals have dependencies (stuff they need to happen)
◇ Systemd tries to satisfy the goal's dependencies to complete the goal
