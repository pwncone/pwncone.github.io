#References


A C++ ‘reference’ is an alias for a variable or object.
At declaration, a reference must be initialised with a variable/object to refer to.

You declare a reference using & after the data type
Traditionally you name a reference the same as the variable/object you're referring, but with its first letter capitalised and with r prepended to the front

int num = 5;
int& rNum = num;


The reference now acts as an alternative name for what it's referring to.
If the reference gets manipulated, what gets changed is the variable/object it's referring to (not the reference itself).
A reference is such a true alias that if you query its memory address it will return the memory address of the variable/object it's referring to. There's no way to discover the memory address of the reference itself.

Beware
Once a reference has been created, it will always refer to the item to which it was initialised - a different item cannot be assigned to that reference.

#include <iostream>
using namespace std;

int main()
{
    int num = 200;
    int& rNum = num;

    cout << num << endl;
    cout << rNum << endl; // will print out the value of num (200)

    // memory address of num
    cout << &num << endl;
    // memory address of num again (because references don't print out their own memory address)
    cout << &rNum << endl;

    return 0;
}


References and Functions


You can also pass references to functions just like you can pointers (refer to Pointers and Functions for example)
