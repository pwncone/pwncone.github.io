## PHP Escaping


The PHP parsing engine needs a way to differentiate PHP code from other elements in the page.
Is called 'escaping'.
4 ways:

Canonical
<php...?>
Universal method. Will always work.

Short-open (SGML-style) tags
<?...?>
Shortest option.
For it to work:
• set --enable-short-tags when building PHP
• set short_open_tag in php.ini to on

ASP-style tags
<%...%>
Copies what ASP does.
Enable via the option in php.ini file.

HTML script tags
<script language = "PHP">...</script>

## Printing


single
print "single line";

multiple lines
print "this spans
multiple lines. The newlines will be
output as well";

print <<<END
This uses the "here document" syntax to output
multiple lines with $variable interpolation. Note
that the here document terminator must appear on a
line with just a semicolon no extra whitespace!
END;

## Whitespace


• PHP is whitespace insensitive
◇ i.e. 1 whitespace character is the same as 10 whitespace characters (it ignores extra)

## Case Sensitive


Variables are case sensitive.
e.g.
$number = 20
print("my number is $number"); - will print 20
print("my number is $nUmBeR"); - won't print anything because variable not recognised
