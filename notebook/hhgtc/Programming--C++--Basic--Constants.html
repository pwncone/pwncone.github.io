#Constants


Data that will not change during the execution of a program should be stored in a constant

Declaring/initialising


A constant can be created for any data type by using the const keyword
Typically, constant names appear in upper case to disinguish them from non-constant variables
const <data-type> <VARIABLE-NAME> = <data>

Unlike variables, constants must always be initialised in the declaration
const int NUMBER = 123456;


enum


The enum keyword provides a handy way to create a sequence of integer constants in a concise manner.
You can name the sequence after the enum keyword.
enum colours {RED, YELLOW, BLUE, GREEN, BLACK, BROWN, GREY};


Unless specified, the first constant will have a value of 0.
Each of the constants will, by default, have a value one greater than the preceeding constant value in the list
(so red will be 0, yellow = 1, blue = 2, green = 3, black = 4, brown = 5, grey = 6 etc.)

To use the constant, you reference its name
cout << “red is worth: ” << RED << endl;


typedef


It can be convenient to define a list of enum constants as a “custom data type” - by using the typedef keyword
The chosen name is added at the end of the enum declaration
typedef enum {NEGATIVE, POSITIVE} electricCharge;

(negative will be 0, positive will be 1)

You can then decalre variables using the custom data type as you would normally
electricCharge neutral = NEGATIVE;
electricCharge active = POSITIVE;


They can also be used to let you create aliases for existing types
e.g. define counter as an int
typedef int counter;
counter value = 100;
