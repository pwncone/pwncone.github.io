## Extend Community License without Signing In


After 30 days your "trial" will end and you'll have to sign in with a Microsoft account to extend it.

PowerShell script available here to extend the license by 31 days.
https://github.com/beatcracker/VSCELicense

1. Run PowerShell as Admin
2. Import-Module -Name .\VSCELicense.psd1
3. Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
4. Set-VSCELicenseExpirationDate -Version VS2019 - extend by 31 days

Apparently the license key is stored in the registry around here somewhere:
HKEY_CLASSES_ROOT\Licenses\5C505A59-E312-4B89-9508-E162F8150517\
More info: https://stackoverflow.com/a/51570570

## Debugging


Set a breakpoint somewhere.
F5 to run in debug mode.
Can debug in both Debug mode and Release (I think)

To view Registers/Memory etc., start debugging then go to Debug > Windows > Registers

## Workspace vs. Projects


Visual C++ differentiates between "workspaces" and "projects".
Project - Involes the creation of a .exe or .dll
Workspace - Contains 1 or more projects
