## BIOS


• The BIOS loads the bootloader by reading the Partition Table
• BIOS is most commonly used with MBR partition table
• If you use BIOS with GPT and want to use the GRUB bootloader, you have to make a BIOS boot partition

### BIOS and MBR (Master Boot Record)


• The Master Boot Record (MBR) is stored on the first 512 bytes of the disk
◇ (it's not located in a partition, it's in the first sector of the device- physical offset 0, before the 1st partition)
• MBR 512 byte breakdown:
◇ first 440 bytes = the stage 1 bootloader (just bootcode)
◇ the remaining 72 bytes = the MBR paritition table
▪ The MBR gets loaded into RAM, at which point the BIOS hands over control to the MBR

Stage 1 Bootloader
1. The stage 1 boot loader looks through the partition table for a stage 2 boot loader and loads it
◇ (the program is only 440 bytes in size, so it has limited functionality due to its size)

Stage 2 Bootloader
1. The stage 2 boot loader (e.g. GRUB) loads the kernel into RAM
◇ (the stage 2 boot loader is stored in a partition)

## UEFI


• UEFI performs EFI booting
• UEFI was designed to be used with a GPT partition table

###

UEFI and GPT (GUID Partition Table)


• The bootlader (GRUB) is stored in the EFI System Partition - called the ESP
• The ESP is FAT32 and 550MiB in size
