# Linux - Filesystem - symlinks & hardlinks


## symlinks


• symlinks are the Linux equivalent to Windows shortcuts
• symlinks allow you to create a link/"shortcut" to a file
• When you modify a symlinked file, the original file also gets modified

ln -s bankDetails bankDetailsLINK - create a symlink

pete@icebox:~/Desktop$ ls -li
total 12
93402 -rw-rw-r-- 1 pete pete 8 Jan 21 21:36 bankDetails
93403 lrwxrwxrwx 1 pete pete 6 Jan 21 21:39 bankDetailsLINK -> bankDetails


## hardlinks


• hardlinks create another file with a link to the same inode
• If you modify the contents of myfile2 or myhardlink, the change would be seen on both,
• If you deleted myfile2, the file would still be accessible through myhardlink

ln somefile somefileHARDLINK - create a hardlink

pete@icebox:~/Desktop$ ln myfile2 myhardlink
pete@icebox:~/Desktop$ ls -li
total 16
  151 -rw-rw-r-- 1 pete pete 7 Jan 21 21:36 myfile
93401 -rw-rw-r-- 2 pete pete 8 Jan 21 21:36 myfile2
93402 -rw-rw-r-- 1 pete pete 8 Jan 21 21:36 myfile3
93403 lrwxrwxrwx 1 pete pete 6 Jan 21 21:39 myfilelink -> myfile
93401 -rw-rw-r-- 2 pete pete 8 Jan 21 21:36 myhardlink
