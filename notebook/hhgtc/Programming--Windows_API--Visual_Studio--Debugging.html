# Debugging



## Attach to process


If you're debugging a DLL in another process,
you can start the DLL running and use Debug > Attach to process to connect to it.
Now you can set breakpoints in your DLL code in Visual Studio and they'll be hit

Example:
https://gamehacking.academy/lesson/28

## system("pause") and Attach to process


If you're debugging an exe that gets modified after compile time,
to debug that .exe you could:
• insert a system("pause > nul") at the start of the program,
• start the program, it should pause
• in Visual Studio, go to Debug > Attach to process and attach
• press ENTER in your program to continue execution

Now you can set breakpoints in your code and debug it
as if you pressed F5 in Visual Studio.
