#Constructor and Destructor Methods


This looks like the preferred/better way to create objects IMO. Do this every time.

Constructor methods
Class variable members can be initialised by a special ‘constructor’ method that's called whenever an instance of the class is created.
The constructor method should always be named the same as the class.

Destructor methdos
When a constructor method is declared, an assosciated ‘destructor’ method should also be declared - this is called whenever a class is destroyed.
The destructor method is named the same as the class, with a ~ prefix attached

Whenever the class is destroyed, the ‘destructor’ code block will run.

#include <iostream>
#include <string>
using namespace std;

class Dog
{
private:
    // CLASS-ONLY DOG VARIABLES
    int age, weight;
    string colour;
public:
    Dog(int, int, string);
    ~Dog();

    // GETTER METHODS
    int getAge() {return age;}
    int getWeight() {return weight;}
    string getColour() {return colour;}

    // PROGRAM-WIDE DOG FUNCTIONS
    void bark() {cout << "WOOF!" << endl;}
};

// Dog - constructor method
Dog::Dog(int age, int weight, string colour)
{
    this ->age = age;
    this ->weight = weight;
    this ->colour = colour;
}

// Dog - destructor method
Dog::~Dog()
{
    cout << "Object destroyed." << endl;
}


int main()
{
    Dog fido(6, 40, "brown");
    Dog pom(10, 15, "grey");

    cout << "Fido is " << fido.getAge() << " years old." << endl;
    cout << "Fido is " << fido.getWeight() << " pounds heavy." << endl;
    cout << "Fido is a " << fido.getColour() << " dog." << endl;

    cout << endl;
    fido.bark();
    cout << endl;

    cout << "Pom is " << pom.getAge() << " years old." << endl;
    cout << "Pom is " << pom.getWeight() << " pounds heavy." << endl;
    cout << "Pom is a " << pom.getColour() << " dog." << endl;
    
    cout << endl;
    pom.bark();
    cout << endl;

    return 0;
}


Constructor and deconstructor methods have no return value, and are called automatically - they cannot be called explicity.
