# Linux - User Management - /etc/sudoers


/etc/sudoers lists all the users that can sudo, and which commands they can run with it.
Usually only the root user can look at this file.
The safe way to edit /etc/sudoers is with visudo

Good explanation - https://www.garron.me/en/linux/visudo-command-sudoers-file-sudo-default-editor.html

## Explanation of the /etc/sudoers file


This is what /etc/sudoers might look like:
# /etc/sudoers
#
# This file MUST be edited with the 'visudo' command as root.
#
# See the man page for details on how to write a sudoers file.
#

Defaults        env_reset

# Host alias specification

# User alias specification

# Cmnd alias specification

# User privilege specification
holly ALL=NOPASSWD:cat


Almost all lines are commented out.

The line that matters in this sudoers file example is:
holly ALL=NOPASSWD:/usr/bin/vi /var/www/html/*
This line means:
holly can run the cat binary with sudo from ALL terminals without providing a password
