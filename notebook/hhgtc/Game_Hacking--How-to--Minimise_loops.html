# Minimise loops


Most cheat devs hook a frame and then call their Aimbot / ESP function.
In each of these functions they loop through all the players.

For example:
void Aimbot()
{
// Loop through all players
for (int i = 0; i < max_players; i++)
{
AimAtPlayer(i);
}
}

void ESP()
{
// Loop through all players
for (int i = 0; i < max_players; i++)
{
DrawBoxAroundPlayer(i);
}
}

void hooked_EndScene()
{
Aimbot();
ESP();

return orig_EndScene()
}


This is excessive, as it means you loop through the entire set of players in the game
multiple times for each of your cheat functions.
In the example above, we're looping through the entire set of players in the game twice.
With more cheat functions, this would increase.

## Solution #1


Instead, you can just execute your cheat code inside of the hooked frame function.
However, it can get a little messy depending on how big your code is.
This is useful for small cheats.
void hooked_EndScene()
{
for (int i = 0; i < max_players; i++)
{
AimAtPlayer(i);
DrawBoxAroundPlayer(i);
}

return orig_EndScene()
}


## Solution #2


Alternatively, you loop once in your hooked frame
and then pass the entitiy or entity index to your Aimbot / ESP function.
This avoids you having to loop through the players in your Aimbot / ESP function
because you've already got the entity to aimbot at or draw a box around.

Example here:
https://www.unknowncheats.me/forum/509115-post7.html

You execute calculations multiple times (WorldToScreen, GetEntityByIndex, etc.)
but achieve cleaner looking code as a result.

void Aimbot(int entity_index)
{
entity = GetEntityByIndex(entity_index);
AimAtEntity(entity)
}

void ESP(int entity_index)
{
entity = GetEntityByIndex(entity_index);
DrawBoxAroundEntity(entity)
}

void hooked_EndScene()
{
for (int i = 0; i < max_players; i++)
{
Aimbot(i);
ESP(i);
}

return orig_EndScene()
}
