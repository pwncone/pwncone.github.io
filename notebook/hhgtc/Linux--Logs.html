# Linux - Logs


• Services, kernel, daemons, etc on your system are constantly doing something
◇ This activity is saved in logs
• Logs are usually kept in /var
• There are no rules to logging, some programs will log whatever they want, wherever they want

## syslog Service


• The syslog service sends system activity to their appropriate log files
• Syslog contains many components
◇ One important component is a daemon called syslogd or rsyslogd
◇ syslogd/ryslogd waits for event messages
▪ it filters these event messages for messages it's interested in
▪ depending on what it's supposed to do with that message, it will:
- send the message to a file
- send the message to your console
- or do nothing with it

less /var/log/syslog - view syslog logs
logger -s Hello - manually send a log

## General Logs


/var/log/messages
◇ contains all non-critical and non-debug messages (useful to get a glimpse of how your machine is acting)
/var/log/syslog
◇ logs everything except auth messages
◇ extremely useful for debugging errors on your machine

## Kernel Logs


/var/log/dmesg
◇ shows information about hardware drivers, kernel information and status during bootup
/var/log/kern.log
◇ logs kernel information and events on your system

## Authentication Logs


/var/log/auth.log
◇ contains system authorization logs, such as user login and the authentication method used

## Log Management


• The logrotate utility manages logs
• Its config file allows you to specify
◇ how many logs to keep
◇ what logs to keep
◇ how to compress logs to save space
• The config files can be found in /etc/logrotate.d
• logrotate is usually run out of cron once a day
