# Linux - Permissions - Set User ID


Files/binaries with a SUID bit (set user ID) allows a user to run a program with the permissions of the user who owns it.
e.g. root owns the ping binary, when a standard user runs the ping binary, it'll run with root permissions.

Add a SUID bit to a file:
chmod +s <file>
chmod 4755 <file> - the numerical representation for SGID is 4

The s denotes the SUID bit.
The binary/file might also be highlighted
-rwsr-xr-- 1 bob bob 18 Feb 24 20:46 suid.txt

## Directories & the SUID bit


On directories the SUID bit works differently
Files created in +s directories receive the permissions of the directory's group (not the user that created it)
