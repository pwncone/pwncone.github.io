# Cron Jobs


Cron jobs schedule and run jobs on the system.
There's always cron jobs on a system.

Cron jobs usually run as root.
If you can tamper with the job itself, or a binary that the cron job uses, you might be able to execute commands as root.

Make sure to searchsploit/google any suspicious/interesting cronjobs you find,
there might be vulnerable versions.

## Check cronjobs


cat /etc/crontab - system wide crontab (i think)
crontab -l - check if your current user has their own crontab
ls -alh /etc/cron* - overview of scheduled jobs (monthly, daily, hourly etc.)

crontab -l <username> - jobs for a specific user
top - currently running tasks

ls -aRl /etc/cron* | awk '$1 ~ /w.$/' 2>/dev/null - what can ‘others’ write in /etc/cron* directories
find / -perm -2 type f 2>/dev/null | grep <file in cron.d> - check whether file is world-writable
