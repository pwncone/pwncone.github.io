# Port 22 - SSH


SSH is always useful to find.
And it's almost always open!

## Checklist


• Grab banner/version
◇ ssh <target ip>
◇ nmap -sC -sV <target ip> -p 22
◇ nc 192.168.1.10 22 -v
• Check for vulns
◇ Debian SSH might be vulnerable to OpenSSL PRNG exploit
• OpenSSH 7.7 and below? Try username enumeration
• Login
◇ Do you have any SSH keys do get in with? Retrieve one from somewhere?
◇ Try common passwords
▪
root
admin
toor
*username in reverse*
*hostname*
*ctf name*
*empty password*

◇ Try every user with every password you've found
◇ If vendor found - Try default credentials
• Bruteforce
◇ usernames you know + top 100
◇ common usernames + top 100

## Config files


/home/user/.ssh/id_rsa - private key
/home/user/.ssh/id_rsa.pub - public key
/home/user/.ssh/authorized_keys - public keys that can authenticate without password
/home/user/.ssh/known_hosts - hosts that have connected
/etc/ssh/ssh_config
/etc/ssh/sshd_config
