# RDP RestrictedAdmin Mode


Implemented in Windows 8.1 and Server 2012 R2.
Designed to mitigate pass-the-hash attacks (by not storing hashes everywhere).

https://blog.ahasayen.com/restricted-admin-mode-for-rdp/

When you RDP into a machine, your credentials are stored on the machine you RDP'd in to.
It's often SysAdmins that RDP into other machines.
Therefore, it's not safe to be storing Admin credentials on all the machines they RDP in to.

Instead, you can use Restricted Admin mode for RDP.
Log in with - mstsc.exe /RestrictedAdmin
You will RDP in, but you credentials wont' be stored on the remote machine.

BECAUSE CREDENTIALS AREN'T STORED,
accessing other network resources from the RDP machine will be done as the current machine (not SysAdmin level)
