#Namespaces


Namespaces are used to avoid name conflicts with functions.
Say you have a function called xyz(), and in another library there's also a function called xyz()
The compiler will have no idea which xyz() function you're trying to refer to.

Namespaces help you avoid these name conflicts.

#include <iostream>
using namespace std;

// create 1st namespace
namespace first_namespace
{
    void xyz()
    {
        cout << "xyz() from first namespace" << endl;
    }
}

// create 2nd namespace
namespace second_namespace
{
    void xyz()
    {
        cout << "xyz() from second namespace" << endl;
    }
}

int main()
{
    first_namespace::xyz();
    second_namespace::xyz();

    return 0;
}


The

using

directive


You can avoid prepending namespaces with the using directive
#include <iostream>
using namespace std;

// create 1st namespace
namespace first_namespace
{
    void xyz()
    {
        cout << "xyz() from first namespace" << endl;
    }
}

// create 2nd namespace
namespace second_namespace
{
    void xyz()
    {
        cout << "xyz() from second namespace" << endl;
    }
}

using namespace second_namespace;
int main()
{
    // will call function from second_namespace    
    xyz();

    return 0;
}


Nested namespaces


#include <iostream>
using namespace std;

// first name space
namespace first_space {
   void func() {
      cout << "Inside first_space" << endl;
   }
   
   // second name space
   namespace second_space {
      void func() {
         cout << "Inside second_space" << endl;
      }
   }
}

using namespace first_space::second_space;
int main () {
   // This calls function from second name space.
   func();
   
   return 0;
}


Use just 1 function from a namespace


The using directive can also be used to refer to a particular item within a namespace.
e.g. if the only part of the std namespace that you want to use is cout, specify that.

#include <iostream>
using std::cout;

int main () {
   cout << "std::endl is used with std!" << std::endl;
   
   return 0;
}
