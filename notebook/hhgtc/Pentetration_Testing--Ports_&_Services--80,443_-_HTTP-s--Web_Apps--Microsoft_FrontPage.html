# Microsoft FrontPage


FrontPage is a WYSIWYG HTML editor and web admin tool.
It allowed users to modify basic and advanced parts of a website with little to no web admin experience.

FrontPage has been around for a while and is notorious for being vulnerable.

• https://dokumen.tips/documents/vtifpxploitation.html - BlackHat talk
• https://insecure.org/sploits/Microsoft.frontpage.insecurities.html - Nmap
• http://www.ouah.org/hfag.txt - A ‘How to Hack FrontPage’ from 1999

## Tips


• Run Nikto against the site, it's good at finding FrontPage vulnerabilities

## Identify FrontPage


You can usually identify Microsoft FrontPage by viewing the page's source and checking the meta tags
<meta name="GENERATOR" content="Microsoft FrontPage 12.0">
<meta name="ProgId" content="FrontPage.Editor.Document">


## Vulnerabilities


### Anonymous FrontPage Login


Older default configurations of FrontPage extensions allowed remote users to login anonymously.

#### Nmap
There's an Nmap script to check for this.
https://nmap.org/nsedoc/scripts/http-frontpage-login.html
nmap <target> -p 80 --script=http-frontpage-login

#### Metasploit
There's also a metasploit module for this.
use auxiliary/scanner/http/frontpage_login

## Recon


### _vti_inf.html


This file provides configuration information.
You can determine the operating system on which FrontPage is running based on references in this file.
• References to .exe files? The OS is probably Unix
• References to .dll files? The OS is probably Windows

### Password files


Account names and passwords are stored in files in the /_vti_pvt directory.
• On Microsoft Web Servers - the password file is service.pwd
• On Netscape Web Servers - the password file is administrators.pwd, authors.pwd or users.pwd

The _vti_pvt directory will probably look like this:
_vti_pvt
    access.cnf
    botinfs.cnf
    service.cnf
    service.pwd ← this is the password file
    writeto.cnf


#### service.pwd
The contents of service.pwd will look like this:
# -FrontPage-
admin:YbV1JnafKRmnQ


admin is the username
YbV1JnafKRmnQ is the password in DES form.

You can bruteforce this hash to get the plaintext password (with a tool like john)

### aspnet_client


This directory often stores scripts.
If no directory listing available, you could bruteforce for files here (with .asp and .aspx file extensions)
