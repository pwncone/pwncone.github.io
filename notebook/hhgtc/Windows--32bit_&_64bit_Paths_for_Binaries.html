# 32bit & 64bit Paths for Binaries


I found these myself when trying to run binaries from 32bit & 64bit processes.

SysNative is a virtual folder used to access the 64bit system32 folder from a 32bit application or script
◇ https://www.samlogic.net/articles/sysnative-folder-64-bit-windows.htm
◇ It's invisible to 64bit in Explorer/CMD/PowerShell
◇ Only 32bit processes will be able to see/access the file

## on 64bit Windows


### from a 64bit CMD


notepad - spawns 64bit notepad
C:\Windows\System32\notepad.exe - Spawns 64bit notepad
C:\Windows\SysWOW64\notepad.exe - Spawns 32bit notepad
C:\Windows\SysNative\notepad.exe - Fails because SysNative doesn't exist

### from a 32bit CMD


notepad - spawns 32bit notepad
C:\Windows\System32\notepad.exe - spawns 32bit notepad
C:\Windows\SysWOW64\notepad.exe - spawns 32bit notepad
C:\Windows\SysNative\notepad.exe - spawns 64bit notepad
