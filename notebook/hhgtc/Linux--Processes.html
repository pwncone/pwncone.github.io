# Linux - Processes


• Each process has an ID associated with it called the process ID (PID)
• PIDs are assigned in the order the processes are created

## Processes & terminals


• Processes are connected to terminals
◇ e.g. you run find and you close the terminal window, the find process goes along with it
• Daemons are not connected to terminals
◇ daemons run in the background and we don't want them to close
◇ for daemons, the TTY column lists them as ? - meaning they don't have a controlling terminal

## Processes & the kernel


• The kernel is in charge of processes
• When you run a program...
◇ the kernel loads up the code of the program in memory
◇ determines and allocates resources to the program
◇ and keeps track of the process
• The kernel knows:
◇ The status of the process
◇ The resources the process is using and receives
◇ The process owner
◇ Signal handling
◇ basically everything

## /proc


• In Linux, everything is a file
• Processes are “files” too
• Every process running will have a “file” entry in /proc
◇ /proc is full of process IDs
◇ cat /proc/12345/status - will show process state information

## ps


ps - list running processes
ps aux
◇ a - display all running processes
◇ u - show more details about the processes
◇ x - list all processes than don't have a TTY

$ ps aux
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root      6126  0.0  0.1   7776  4528 pts/0    Ss   06:28   0:00 bash
root      7633  0.0  0.0   8464  2944 pts/0    R+   18:53   0:00 ps aux


• USER: The effective user (the one whose access we are using)
• PID: Process ID
• %CPU: CPU time used divided by the time the process has been running
• %MEM: Ratio of the process's resident set size to the physical memory on the machine
• VSZ: Virtual memory usage of the entire process
• RSS: Resident set size, the non-swapped physical memory that a task has used
• TTY: Controlling terminal associated with the process
• STAT: Process status code
◇ R = running or runnable, it is just waiting for the CPU to process it
◇ S = Interruptible sleep, waiting for an event to complete, such as input from the terminal
◇ D = Uninterruptible sleep, processes that cannot be killed or interrupted with a signal, usually to make them go away you have to reboot or fix the issue
◇ Z = Zombie, we discussed in a previous lesson that zombies are terminated processes that are waiting to have their statuses collected
◇ T = Stopped, a process that has been suspended/stopped
• START: Start time of the process
• TIME: Total CPU usage time
• COMMAND: Name of executable/command
