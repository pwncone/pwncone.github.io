# Linux - I/O Redirection - File Descriptors


https://www.bottomupcs.com/file_descriptors.xhtml
On Linux, everything is a file. Processes are files, network sockets are files, the keyboard input is a file.
To make it easy for programmers to access these ‘files’, the linux kernel holds a table of all the open ‘files’ on the system and each of them have an assigned number.

For example:
• 4 = libreoffice document
• 5 = indianajones.mp4
• 6 = research_notes.txt

The kernel will assign a file descriptor to a file whenever one gets opened.
However, some ‘files’ are always open and always have a file descriptor - stdin, stdout and stderr

The first 3 file descriptors on Linux are always:
• 0 = stdin
• 1 = stdout
• 2 = stderr
